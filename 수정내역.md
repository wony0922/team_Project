# DB-Buddy 수정 내역

## 요약

로컬에서 저장소를 내려받아 실행되는지 확인했고, `localhost` 전용이 아니라 같은 네트워크에서 접속 가능한 주소로 실행되도록 수정했습니다.

실행 검증 결과:

- Python 문법 검사 통과
- SQLite 샘플 DB 초기화 성공
- Streamlit 서버 실행 성공
- `0.0.0.0:8501` 바인딩 확인
- `http://127.0.0.1:8501` 응답 `200 OK` 확인
- Streamlit 출력에서 `Network URL: http://42.82.208.27:8501` 확인

## 수정 파일

### `.gitignore`

- [수정됨] `DB_Buddy/.venv/`를 Git 추적 제외에 추가했습니다.
- [수정됨] 루트 `.venv/`도 Git 추적 제외에 추가했습니다.
- [수정됨] 로컬 에디터 설정 폴더 `.vscode/`도 Git 추적 제외에 추가했습니다.

수정 이유:

- 실행 검증 과정에서 만든 가상환경이 GitHub에 올라가지 않도록 하기 위함입니다.
- 개인 로컬 에디터 설정이 커밋에 섞이지 않도록 하기 위함입니다.

### `DB_Buddy/bootstrap.py`

- [수정됨] pip 패키지명과 import 모듈명이 다른 패키지를 정확히 검사하도록 변경했습니다.
- [수정됨] `streamlit-mermaid`는 `streamlit_mermaid` 모듈로 확인하도록 수정했습니다.
- [수정됨] `langchain-community`는 `langchain_community` 모듈로 확인하도록 수정했습니다.
- [수정됨] `PyMySQL` 의존성 확인을 추가했습니다.
- [수정됨] Streamlit 실행 시 기본 주소를 `0.0.0.0`으로 바인딩하도록 변경했습니다.
- [수정됨] `DB_BUDDY_HOST`, `DB_BUDDY_PORT` 환경변수로 실행 주소와 포트를 바꿀 수 있게 했습니다.

수정 이유:

- 기존 방식은 일부 패키지를 잘못 누락 판단할 수 있었습니다.
- 기본 Streamlit 실행이 `localhost` 중심이라 다른 기기에서 접속하기 어려웠습니다.

### `DB_Buddy/db_utils.py`

- [수정됨] SQLite DB 파일 경로를 `DB_Buddy/local_study.db`로 고정했습니다.
- [수정됨] `sample_data.sql`을 현재 실행 위치가 아니라 `DB_Buddy` 폴더 기준으로 읽도록 수정했습니다.
- [수정됨] DB 초기화와 DB 복원 기능 모두 같은 경로 기준을 사용하게 했습니다.

수정 이유:

- 루트에서 실행할 때 `sample_data.sql`을 찾지 못하거나 DB가 엉뚱한 위치에 생성될 수 있었습니다.

### `DB_Buddy/desktop_app.py`

- [수정됨] 데스크톱 앱 실행 시 `DB_BUDDY_HOST`, `DB_BUDDY_PORT` 환경변수를 지원하도록 했습니다.
- [수정됨] Streamlit 서버 실행 포트와 pywebview 접속 포트를 동일하게 맞췄습니다.

수정 이유:

- 데스크톱 실행 방식에서도 포트 변경과 외부 바인딩 설정을 사용할 수 있게 하기 위함입니다.

### `DB_Buddy/requirements.txt`

- [수정됨] `PyMySQL>=1.1.0`을 추가했습니다.

수정 이유:

- `app.py`의 MySQL 연결 기능에서 `mysql+pymysql://...`을 사용하지만, 기존 의존성 파일에는 `PyMySQL`이 빠져 있었습니다.

### `DB_Buddy/app.py`

- [수정됨] SQLite 초기화 확인 경로를 `DB_Buddy/local_study.db` 기준으로 변경했습니다.
- [수정됨] MySQL 연결 시 입력한 데이터베이스가 없으면 자동으로 `CREATE DATABASE IF NOT EXISTS`를 실행하도록 변경했습니다.
- [수정됨] MySQL 비밀번호에 특수문자가 들어가도 연결 문자열이 깨지지 않도록 SQLAlchemy `URL.create()`를 사용했습니다.
- [수정됨] MySQL 데이터베이스 이름은 영문, 숫자, `_`, `$`만 허용하도록 검사했습니다.

수정 이유:

- `test_db`가 아직 생성되지 않은 상태에서 바로 `mysql+pymysql://.../test_db`로 접속해 `(1049, "Unknown database 'test_db'")` 오류가 발생했습니다.
- 이제 MySQL 서버에 먼저 접속한 뒤 DB를 만들고, 그 다음 해당 DB로 연결합니다.

### `README.md`

- [수정됨] `localhost`가 아닌 네트워크 주소로 접속하는 방법을 추가했습니다.
- [수정됨] `DB_BUDDY_HOST`, `DB_BUDDY_PORT` 환경변수 사용 예시를 추가했습니다.

## 실행 방법

저장소 루트에서 다음 파일을 실행합니다.

```bat
runFile!!!!!!!!!!!!.bat
```

기본 실행 주소:

```text
http://localhost:8501
```

같은 네트워크의 다른 기기에서 접속:

```text
http://<실행한-PC-IP>:8501
```

호스트와 포트를 직접 지정:

```bat
set DB_BUDDY_HOST=0.0.0.0
set DB_BUDDY_PORT=8501
runFile!!!!!!!!!!!!.bat
```

## 참고

현재 작업 PC에는 기본 `python` 명령이 잡혀 있지 않아, 실행 검증을 위해 `DB_Buddy/.venv` 가상환경을 만들었습니다. 이 폴더는 `.gitignore`에 추가되어 Git에는 올라가지 않습니다.
